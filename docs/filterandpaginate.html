Documentation Plugin FilterAndPaginate

Objectif

Le plugin FilterAndPaginate ajoute deux fonctionnalités essentielles aux tableaux HTML gérés par TableFlow :

  • Filtrage Global : Permet de filtrer les lignes du tableau en fonction d'un terme de recherche saisi dans un champ de texte externe. La recherche s'effectue sur l'ensemble des cellules visibles de chaque ligne.
  • Pagination Côté Client : Divise le contenu du tableau (les lignes filtrées) en plusieurs pages, avec des contrôles de navigation (premier, précédent, suivant, dernier, numéros de page) et éventuellement un sélecteur pour choisir le nombre de lignes par page.

Ce plugin fonctionne entièrement côté client, ce qui signifie qu'il opère sur les données déjà présentes dans le DOM du tableau. Il est idéal pour des tableaux de taille modérée où charger toutes les données initialement est acceptable.

Activation et Configuration HTML

Ce plugin ne nécessite aucun attribut spécifique sur les éléments <th> pour son fonctionnement de base. Son activation et sa configuration se font principalement via JavaScript.

Cependant, vous devez fournir un élément <input> quelque part dans votre page HTML si vous souhaitez utiliser la fonctionnalité de filtrage global. L'ID ou le sélecteur CSS de cet input sera passé dans la configuration JavaScript. 

<!-- Input pour le filtre global (en dehors de la table) -->
<input type="text" id="monFiltreGlobal" placeholder="Filtrer le tableau...">

<!-- Votre table HTML -->
<table id="maTable">
  <thead>
    <tr>
      <th id="col1">Colonne 1</th>
      <th id="col2">Colonne 2</th>
      <!-- ... -->
    </tr>
  </thead>
  <tbody>
    <tr id="row1">
        <td id="col1_row1">Texte A</td>
        <!-- Utiliser data-filter-value pour une valeur de filtre différente du texte affiché -->
        <td id="col2_row1" data-filter-value="valeurCachee1">Label B</td>
    </tr>
     <tr id="row2">
        <td id="col1_row2">Texte C</td>
        <td id="col2_row2">Label D</td> <!-- Filtrera sur "Label D" -->
    </tr>
    <!-- ... autres lignes -->
  </tbody>
</table>

<!-- Le conteneur de pagination sera inséré ici par le plugin -->

Pour le filtrage, le plugin recherche le terme saisi dans le contenu textuel de chaque cellule (``). Vous pouvez fournir une valeur de filtrage alternative pour une cellule en ajoutant l'attribut data-filter-value à l'élément ``. Si cet attribut est présent, sa valeur sera utilisée pour le filtrage au lieu du contenu textuel de la cellule.

Configuration JavaScript

La configuration du plugin se fait dans l'objet d'options de `TableFlow`, sous la clé plugins.filterandpaginate. 

const tableInstance = new TableFlow({
    tableId: 'maTable',
    pluginsPath: '../plugins', // Ajustez
    plugins: {
        names: ['FilterAndPaginate', /* ... autres plugins ... */], // Activer le plugin
        filterandpaginate: { // Configuration spécifique
            // --- Options de Filtrage ---
            enableFilter: true, // Activer/désactiver le filtrage (défaut: true)
            globalFilter: '#monFiltreGlobal', // Sélecteur CSS de l'input de filtre (requis si enableFilter=true)
            debounceTime: 300, // Délai (ms) après saisie avant de filtrer (défaut: 300)

            // --- Options de Pagination ---
            pageSize: 10, // Nombre de lignes par page (défaut: 10)
            pageSizes: [10, 25, 50, 100], // Options pour le sélecteur de taille (défaut: [10, 25, 50, 100])
            showPageSizes: true, // Afficher le sélecteur de taille de page (défaut: true)
            showInfo: true, // Afficher l'info "Affichage X à Y sur Z" (défaut: true)

            // --- Classes CSS ---
            containerClass: 'pagination-container', // Classe du conteneur global (défaut)
            paginationClass: 'pagination', // Classe du conteneur des boutons de page (défaut)
            activeClass: 'active', // Classe pour le bouton de la page active (défaut)
            disabledClass: 'disabled', // Classe pour les boutons désactivés (préc/suivant...) (défaut)
            selectClass: 'form-select', // Classe pour le
