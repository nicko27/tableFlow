Documentation Plugin ContextMenu

Objectif

Le plugin ContextMenu fournit un système centralisé pour afficher un menu contextuel personnalisé (menu clic droit) sur les cellules (`<td>`) d'un tableau TableFlow.

Il ne définit pas lui-même les options du menu. Au lieu de cela, il agit comme un **gestionnaire** qui interroge d'autres plugins actifs (les "fournisseurs" ou "providers") pour savoir quelles options de menu proposer pour la cellule spécifique sur laquelle l'utilisateur a fait un clic droit.

Des plugins comme TextEditor ou Selection peuvent s'enregistrer auprès de ContextMenu pour ajouter leurs propres actions au menu contextuel.

Activation et Configuration HTML

Ce plugin ne nécessite aucun attribut HTML spécifique pour son activation. Il est activé et configuré uniquement via JavaScript.

Configuration JavaScript

La configuration du plugin se fait dans l'objet d'options de `TableFlow`, sous la clé plugins.contextmenu. 

const tableInstance = new TableFlow({
    tableId: 'maTable',
    pluginsPath: '../plugins', // Ajustez
    plugins: {
        names: ['ContextMenu', /* ... autres plugins fournisseurs comme Edit, TextEditor, Selection ... */], // Activer le plugin
        contextmenu: { // Configuration spécifique pour 'ContextMenu'
            menuClass: 'tf-context-menu', // Classe CSS pour le conteneur du menu (défaut)
            menuItemClass: 'tf-menu-item', // Classe CSS pour chaque option du menu (défaut)
            menuSeparatorClass: 'tf-menu-separator', // Classe CSS pour les séparateurs (défaut)
            menuHeaderClass: 'tf-menu-header', // Classe CSS pour les en-têtes de section (défaut)
            debug: false // Activer les logs spécifiques
        }
        // ... configuration des autres plugins ...
    }
    // ... autres options TableFlow ...
});

Interface Fournisseur (Provider Interface)

Pour qu'un autre plugin puisse ajouter des options au menu contextuel, il doit s'enregistrer auprès du plugin ContextMenu et implémenter une interface spécifique.

  1. Enregistrement : Dans la méthode `init()` du plugin fournisseur, après avoir obtenu une référence au plugin ContextMenu, il doit appeler : 
    // Dans le plugin fournisseur (ex: TextEditor)
init(tableHandler) {
    this.table = tableHandler;
    this.contextMenuPlugin = this.table.getPlugin('ContextMenu');
    if (this.contextMenuPlugin) {
        this.contextMenuPlugin.registerProvider(this); // 'this' est l'instance du plugin fournisseur
    }
    // ...
}
  2. Méthode `getMenuItems(cell)` : Le plugin fournisseur DOIT implémenter cette méthode. Elle est appelée par ContextMenu lors d'un clic droit.
    • Elle reçoit en argument l'élément `<td>` sur lequel le clic droit a eu lieu.
    • Elle DOIT retourner un tableau d'objets représentant les options de menu que ce fournisseur souhaite ajouter pour cette cellule spécifique. Si aucune option n'est pertinente, elle retourne un tableau vide `[]`.
  3. Méthode `executeAction(actionId, cell)` : Le plugin fournisseur DOIT implémenter cette méthode. Elle est appelée par ContextMenu lorsque l'utilisateur clique sur une option de menu fournie par ce plugin.
    • Elle reçoit l'`id` de l'action (tel que défini dans l'objet retourné par `getMenuItems`) et la cellule `<td>` qui était ciblée lors de l'ouverture du menu.
    • Elle doit contenir la logique pour exécuter l'action correspondante.

Structure d'un Objet Menu Item (retourné par `getMenuItems`)

Chaque objet dans le tableau retourné par `getMenuItems` peut avoir les propriétés suivantes :

PropriétéTypeRequis?Description
typestringNonPeut être `'separator'` pour insérer un séparateur ou `'header'` pour un titre de section (utilise `label`). Si omis, c'est une option de menu standard.
idstringOui (si standard)Identifiant unique pour cette action au sein du fournisseur. Sera passé à `executeAction`. Non requis pour `separator` ou `header`.
labelstringOui (si standard ou header)Le texte affiché pour l'option de menu ou l'en-tête de section.
iconstringNonHTML (ou caractère) pour l'icône à afficher avant le label.
actionfunctionNon(Alternative) Une fonction `(cell) => void` à exécuter directement au clic. Si fournie, `executeAction` du fournisseur ne sera pas appelée pour cet item. Moins courant.
providerobject(Interne)Référence au plugin fournisseur, ajoutée automatiquement par ContextMenu.
 
// Exemple de retour de getMenuItems(cell) dans un plugin fournisseur
getMenuItems(cell) {
    const items = [];
    const isEditable = cell.classList.contains('td-edit'); // Exemple de condition

    if (isEditable) {
        items.push({ type: 'header', label: 'Mon Plugin' });
        items.push({ id: 'action1', label: 'Faire Action 1', icon: '🚀' });
        items.push({ id: 'action2', label: 'Faire Action 2', icon: '💡' });
        items.push({ type: 'separator' });
        items.push({ id: 'action3', label: 'Autre Chose' });
    }
    return items;
}

// Exemple de executeAction(actionId, cell) dans le même plugin
executeAction(actionId, cell) {
    if (actionId === 'action1') {
        // Logique pour action 1 sur 'cell'
        console.log('Action 1 sur', cell.id);
    } else if (actionId === 'action2') {
        // Logique pour action 2
        console.log('Action 2 sur', cell.id);
    } else if (actionId === 'action3') {
        // Logique pour action 3
        console.log('Action 3 sur', cell.id);
    }
}

Fonctionnement Interne

  • Initialisation (`init`, `createContextMenu`, `setupEventListeners`): Crée l'élément `div` du menu (caché) et l'ajoute au `body`. Ajoute un écouteur `contextmenu` à la table et des écouteurs `click` et `keydown` au document pour fermer le menu.
  • Enregistrement (`registerProvider`): Ajoute un plugin fournisseur à sa liste interne `providers`.
  • Gestion du Clic Droit (`handleContextMenu`):
    • Empêche le menu navigateur par défaut.
    • Identifie la cellule `` cliquée.
    • Appelle `collectMenuItems(cell)` pour récupérer les options de tous les fournisseurs.
    • Si des options sont retournées, appelle `showMenu(x, y, items)` pour construire et afficher le menu.
  • Collecte des Options (`collectMenuItems`): Itère sur `this.providers`, appelle `provider.getMenuItems(cell)` pour chacun, ajoute un séparateur entre les groupes d'options de différents fournisseurs, et retourne la liste complète.
  • Affichage du Menu (`showMenu`):
    • Vide le contenu précédent du `div` de menu.
    • Itère sur les `items` collectés :
      • Crée un `div` pour chaque item (option, séparateur, header) avec les classes CSS configurées.
      • Ajoute le contenu (label, icône).
      • Pour les options cliquables, ajoute des écouteurs `mouseover`/`mouseout` pour le style de survol et un écouteur `click`.
    • L'écouteur `click` sur une option appelle `this.hideMenu()` puis la méthode `executeAction(itemId, currentCell)` du *fournisseur* associé à l'item (ou la fonction `item.action` si elle existe).
    • Positionne le menu aux coordonnées du clic (`x`, `y`).
    • Ajuste la position si le menu dépasse des bords de la fenêtre.
    • Affiche le menu (`display: 'block'`).
  • Masquage du Menu (`hideMenu`): Cache le `div` du menu (`display: 'none'`) et réinitialise `currentCell`.

API Publique

La principale méthode publique est destinée aux autres plugins pour s'enregistrer :

MéthodeArgumentsDescriptionRetour
registerProvider(provider)provider (object)Enregistre un autre plugin comme fournisseur d'options de menu. Le `provider` doit implémenter `getMenuItems(cell)` et `executeAction(actionId, cell)`.this (l'instance ContextMenu)

Événements

  • Écoutés :
    • contextmenu (sur l'élément ``) : Pour déclencher l'affichage du menu.
    • click (sur `document`) : Pour masquer le menu si un clic a lieu en dehors.
    • keydown (sur `document`) : Pour masquer le menu si la touche Échap est pressée.
    • Déclenchés : Aucun événement spécifique déclenché par ce plugin. Il facilite l'exécution d'actions définies par d'autres plugins.

      • Exemple Complet

      Cet exemple montre comment activer le plugin ContextMenu et comment un plugin simple (ici, un "LoggerPlugin" fictif) peut s'y enregistrer pour ajouter une option.

      HTML (`index.html`): 
      <!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Exemple ContextMenu Plugin</title>
      
    
</head>
<body>
    <h1>TableFlow - Exemple Plugin ContextMenu</h1>
    <p>Faites un clic droit sur une cellule.</p>

    <table id="simpleTable">
        <thead>
            <tr>
                <th id="colA">Colonne A</th>
                <th id="colB">Colonne B</th>
            </tr>
        </thead>
        <tbody>
            <tr id="r1">
                <td id="colA_r1">Valeur 1A</td>
                <td id="colB_r1">Valeur 1B</td>
            </tr>
            <tr id="r2">
                <td id="colA_r2">Valeur 2A</td>
                <td id="colB_r2">Valeur 2B</td>
            </tr>
        </tbody>
    </table>

    <script type="module">
        import TableFlow from '../src/tableFlow.js'; // Ajustez

        // --- Définition d'un plugin fournisseur Fictif ---
        class LoggerPlugin {
            constructor(config = {}) {
                this.name = 'Logger';
                this.table = null;
                this.contextMenuPlugin = null;
            }
            init(tableHandler) {
                this.table = tableHandler;
                this.contextMenuPlugin = this.table.getPlugin('ContextMenu');
                if (this.contextMenuPlugin) {
                    this.contextMenuPlugin.registerProvider(this);
                    console.log('LoggerPlugin enregistré auprès de ContextMenu.');
                }
            }
            // Méthode requise par ContextMenu
            getMenuItems(cell) {
                console.log('LoggerPlugin: getMenuItems appelé pour', cell.id);
                return [
                    { type: 'header', label: 'Logs' },
                    { id: 'logCellId', label: 'Log ID Cellule', icon: '🆔' },
                    { id: 'logCellValue', label: 'Log Valeur Cellule', icon: '💬' }
                ];
            }
            // Méthode requise par ContextMenu
            executeAction(actionId, cell) {
                const value = cell.getAttribute('data-value') || cell.textContent.trim();
                if (actionId === 'logCellId') {
                    alert(`ID de la cellule : ${cell.id}`);
                    console.log(`ID de la cellule : ${cell.id}`);
                } else if (actionId === 'logCellValue') {
                    alert(`Valeur de la cellule ${cell.id} : ${value}`);
                    console.log(`Valeur de la cellule ${cell.id} : ${value}`);
                }
            }
            destroy() { /* Nettoyage si nécessaire */ }
        }
        // --- Fin du plugin fournisseur fictif ---


        // --- Initialisation de TableFlow ---
        const simpleTable = new TableFlow({
            tableId: 'simpleTable',
            pluginsPath: '../plugins', // Ajustez
            debug: true,
            plugins: {
                // Activer ContextMenu et notre LoggerPlugin
                // Note: LoggerPlugin n'est pas un fichier séparé ici,
                // mais dans un vrai projet, il le serait et serait chargé comme les autres.
                names: ['ContextMenu'], // 'Logger' serait ajouté s'il était un fichier séparé
                contextmenu: {
                    // Config par défaut
                }
                // logger: {} // Config pour Logger si nécessaire
            }
        });

        // Enregistrer manuellement notre LoggerPlugin après l'initialisation de TableFlow
        // (Car il n'est pas chargé comme les autres plugins dans cet exemple)
        const loggerInstance = new LoggerPlugin();
        loggerInstance.init(simpleTable); // L'init enregistre le provider

    </script>
</body>
</html>
      Fonctionnement de l'exemple :
      1. La table s'affiche normalement.
      2. Le plugin `ContextMenu` est activé.
      3. Un plugin fictif `LoggerPlugin` est défini et instancié. Sa méthode `init` l'enregistre auprès de `ContextMenu`.
      4. Lorsque vous faites un clic droit sur une cellule (ex: "Valeur 1A") :
        • Le plugin `ContextMenu` intercepte l'événement.
        • Il appelle `getMenuItems()` sur `LoggerPlugin`.
        • `LoggerPlugin` retourne les options "Log ID Cellule" et "Log Valeur Cellule".
        • `ContextMenu` affiche le menu avec ces options sous l'en-tête "Logs".
      5. Si vous cliquez sur "Log ID Cellule", `ContextMenu` appelle `LoggerPlugin.executeAction('logCellId', cell)`.
      6. `LoggerPlugin` affiche alors une alerte avec l'ID de la cellule (ex: "colA_r1").
